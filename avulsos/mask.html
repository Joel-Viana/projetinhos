CSS Masks (mask-*) — Guia completo

Máscara controla visibilidade por pixel (alpha/luminância). Ela não altera layout e não “pinta” cor — ela só decide o que aparece e quanto aparece.

Conceito base: o que é máscara

(máscara ≈ “background da visibilidade”)

Máscara é uma “película” aplicada ao elemento. Por pixel:

  • alpha alto (preto/1) → o pixel do elemento aparece
  • alpha baixo (transparente/0) → o pixel do elemento some
  • meio termo → aparece parcialmente (fade)

Importante: máscara não muda o tamanho do elemento. Ele continua ocupando espaço no layout.

Diferença rápida:

  • opacity → reduz o elemento inteiro (inclui filhos)
  • mask → recorta por pixel (pode ser gradiente/imagem)
  • display:none → remove do layout
/* Regra mental (memorizar)
   Máscara controla "ONDE APARECE" e "QUANTO APARECE".
   Ela não define cor, não define blur, não muda layout. */

1) mask-image

Define a “fonte” da máscara: gradiente(s) e/ou imagem(ns)

Demo — radial-gradient (vinheta)

Centro aparece → bordas somem
Demo — conic-gradient (angular)

/* mask-image: define QUAL é a máscara (gradiente ou imagem)
   - Preto/alpha 1 = visível
   - Transparente/alpha 0 = invisível */

.element {
  mask-image: linear-gradient(to right, black 0%, transparent 100%); /* gradiente como máscara */
  -webkit-mask-image: linear-gradient(to right, black 0%, transparent 100%); /* Safari */
}

/* Valores aceitos (mask-image):
   - none
   - linear-gradient(...)
   - radial-gradient(...)
   - conic-gradient(...)
   - url("mask.svg" | "mask.png")
   - múltiplas máscaras por camada (separadas por vírgula) */

2) mask-size / 3) mask-position / 4) mask-repeat

Controlam tamanho, posição e repetição (igual background-*)

Essas três propriedades funcionam como as equivalentes de background: mask-size define “quanto” da área a máscara cobre, mask-position define “onde” ela começa, e mask-repeat define se a máscara se repete.

Elas são cruciais quando você usa url() (imagem) ou quando quer alinhamento perfeito da máscara. 

.element {
  mask-image: url("mask.svg");              /* máscara por imagem */
  -webkit-mask-image: url("mask.svg");      /* Safari */

  mask-size: contain;                       /* tamanho da máscara */
  -webkit-mask-size: contain;               /* Safari */

  mask-position: center;                    /* posição da máscara */
  -webkit-mask-position: center;            /* Safari */

  mask-repeat: no-repeat;                   /* repetição da máscara */
  -webkit-mask-repeat: no-repeat;           /* Safari */
}

/* Valores aceitos:
   mask-size:
   - auto
   - cover
   - contain
   - <length-percentage>{1,2}  (ex: 100% 100%, 200px 120px)

   mask-position:
   - left | right | top | bottom | center
   - <percentage> (ex: 50% 0%)
   - <length> (ex: 12px 40px)

   mask-repeat:
   - repeat
   - repeat-x
   - repeat-y
   - no-repeat
   - space
   - round */

5) mask-mode

Define se a máscara usa alpha (transparência) ou luminance (brilho)

mask-mode controla como a máscara é “lida”:

  • alpha: usa a transparência do pixel (o mais comum)
  • luminance: usa o brilho (preto/branco) como base
  • match-source: usa o modo “natural” da fonte (depende do tipo)

Na prática: para gradientes e PNG com transparência, alpha costuma ser o esperado. 

.element {
  mask-image: url("mask.png");      /* PNG pode ter alpha e/ou brilho */
  -webkit-mask-image: url("mask.png");

  mask-mode: alpha;                 /* interpreta pela transparência */
  /* mask-mode: luminance; */       /* interpreta pelo brilho */
}

/* Valores aceitos:
   - match-source
   - alpha
   - luminance */

6) mask-origin e 7) mask-clip

Origin = referência / Clip = limite de aplicação
Demo — origin/clip usando content-box
mask-origin / mask-clip
Referência e limite mudam com border/padding 
/* mask-origin: define QUAL "caixa" do box-model é a referência
   para posicionar/dimensionar a máscara (0,0 e área base).

   mask-clip: define ATÉ ONDE a máscara pode "pintar" (limite). */

.element {
  border: 10px solid transparent;                /* existe border-box */
  padding: 20px;                                 /* existe padding-box */

  mask-image: linear-gradient(black, transparent);/* máscara gradiente */
  -webkit-mask-image: linear-gradient(black, transparent);

  mask-origin: content-box;                      /* referência = conteúdo */
  -webkit-mask-origin: content-box;

  mask-clip: content-box;                        /* aplica só no conteúdo */
  -webkit-mask-clip: content-box;

  mask-position: top left;                       /* relativo ao origin */
  -webkit-mask-position: top left;

  mask-size: 100% 100%;                          /* relativo ao origin */
  -webkit-mask-size: 100% 100%;
}

/* Valores aceitos:
   mask-origin:
   - border-box
   - padding-box
   - content-box

   mask-clip:
   - border-box
   - padding-box
   - content-box
   - text */

8) mask-composite

Combina múltiplas máscaras (layers) no MESMO elemento
Demo — duas máscaras + composite
mask-composite: intersect
Combina layers separados por vírgula

Ponto crítico: mask-composite não “junta” máscaras de elementos diferentes.

Ele só funciona quando você define múltiplas camadas de máscara no mesmo elemento usando vírgulas:

  • mask-image: layer1, layer2, layer3;
  • e então mask-composite define como elas se combinam
/* Exemplo: gradient + outro recorte (duas layers) */
.element {
  mask-image:
    linear-gradient(to right, black 0%, transparent 100%),     /* layer 1 */
    radial-gradient(circle at 50% 50%, black 0%, black 55%, transparent 60%); /* layer 2 */
  -webkit-mask-image:
    linear-gradient(to right, black 0%, transparent 100%),
    radial-gradient(circle at 50% 50%, black 0%, black 55%, transparent 60%);

  mask-repeat: no-repeat, no-repeat;                           /* por layer */
  -webkit-mask-repeat: no-repeat, no-repeat;

  mask-size: 100% 100%, 100% 100%;                             /* por layer */
  -webkit-mask-size: 100% 100%, 100% 100%;

  mask-position: center, center;                               /* por layer */
  -webkit-mask-position: center, center;

  mask-composite: intersect;                                   /* combinação */
  /* -webkit-mask-composite: source-in; */                     /* Safari: valores diferem */
}

/* Valores aceitos:
   mask-composite (spec):
   - add
   - subtract
   - intersect
   - exclude

   Observação Safari/WebKit:
   -webkit-mask-composite usa keywords diferentes (comportamento parecido, mas não 1:1). */

9) mask-type

Mais relevante em SVG: define alpha vs luminance

mask-type aparece principalmente quando você trabalha com máscaras definidas em SVG. No CSS “do dia a dia”, muitas vezes você nem precisa tocar nisso.

Ele define se a máscara do SVG deve ser interpretada por luminância ou por alpha. 

.element {
  mask-image: url("mask.svg");       /* máscara SVG */
  -webkit-mask-image: url("mask.svg");

  mask-type: luminance;             /* interpreta brilho no SVG */
  /* mask-type: alpha; */           /* interpreta transparência no SVG */
}

/* Valores aceitos:
   - luminance
   - alpha */

10) Shorthand: mask

Atalho que agrupa várias propriedades em uma linha

mask é shorthand. Ele pode incluir imagem, posição, tamanho e repetição (e mais).

Na prática profissional, muitas vezes é melhor usar propriedades separadas quando você quer clareza e manutenção. 

/* shorthand (forma compacta) */
.element {
  mask: linear-gradient(to bottom, black 0%, transparent 70%) no-repeat center / 100% 100%;
  -webkit-mask: linear-gradient(to bottom, black 0%, transparent 70%) no-repeat center / 100% 100%;
}

/* Componentes comuns dentro do shorthand:
   - mask-image (gradiente/url)
   - mask-repeat
   - mask-position
   - mask-size (após '/')

   Valores aceitos no shorthand dependem das subpropriedades. */

Exemplo completo: “blur em degradê” (sem degradê de cor)

backdrop-filter aplica blur; mask controla onde ele aparece
Demo — blur que some (topo → baixo)
backdrop-filter + mask
Máscara “abre” o blur gradualmente

Importante: você queria o degradê no blur, não na cor. A técnica correta é:

  1. Aplicar backdrop-filter: blur(...) no overlay
  2. Aplicar mask-image: linear-gradient(...) no overlay

Isso faz o blur “sumir” sem você precisar fazer degradê de cor no background. 

.container {
  position: relative;                  /* base do overlay */
  overflow: hidden;                    /* não deixa o overlay vazar */
}

.container::before {
  content: "";                         /* cria o overlay */
  position: absolute;                  /* overlay por cima */
  inset: 0;                            /* cobre tudo */
  pointer-events: none;                /* não bloqueia clique */

  backdrop-filter: blur(10px);         /* BLUR (uniforme) */
  -webkit-backdrop-filter: blur(10px); /* Safari */

  background: rgba(255,255,255,0.001); /* ativa o backdrop sem cor perceptível */

  mask-image: linear-gradient(to bottom, black 0%, transparent 70%); /* DEGRADÊ DO BLUR */
  -webkit-mask-image: linear-gradient(to bottom, black 0%, transparent 70%);
}

/* Valores aceitos:
   backdrop-filter:
   - none
   - blur(px)
   - brightness(%)
   - contrast(%)
   - grayscale(%)
   - hue-rotate(deg)
   - invert(%)
   - opacity(%)
   - saturate(%)
   - sepia(%)
   - drop-shadow()

   mask-image: none | gradient() | url() */

Compatibilidade e prática profissional

prefixos, fallback e consistência
  • Safari: use sempre -webkit-mask-* junto do padrão.
  • Firefox: suporte pode variar; teste quando for requisito do projeto.
  • Fallback: se máscara não existir, o elemento aparece inteiro (comportamento aceitável em muitos casos).
  • Manutenção: prefira propriedades separadas quando estiver ajustando origin/clip/position/size.

Regra para memorizar: Máscara é “background da visibilidade” → pense em background-*

/* Padrão recomendado (compat):
.element {
  mask-image: linear-gradient(black, transparent);        /* padrão */
  -webkit-mask-image: linear-gradient(black, transparent);/* Safari */

  mask-repeat: no-repeat;
  -webkit-mask-repeat: no-repeat;

  mask-size: 100% 100%;
  -webkit-mask-size: 100% 100%;

  mask-position: center;
  -webkit-mask-position: center;
} */

Resumo final (o que cada coisa faz)

para bater o olho e lembrar
  • mask-image: define a máscara (gradiente/imagem) e suas camadas
  • mask-size: define o tamanho da máscara (auto/cover/contain/medidas)
  • mask-position: define a posição da máscara (center/%/px)
  • mask-repeat: define repetição (repeat/no-repeat/space/round)
  • mask-mode: como interpretar (alpha/luminance)
  • mask-origin: referência (border/padding/content box)
  • mask-clip: limite de aplicação (border/padding/content/text)
  • mask-composite: combina layers no mesmo elemento (add/subtract/intersect/exclude)
  • mask-type: (mais SVG) alpha vs luminance
  • mask: shorthand
/* Resposta estilo prova:
   Máscara (mask-*) controla visibilidade por pixel usando alpha/luminância,
   com propriedades muito parecidas com background-*. */
